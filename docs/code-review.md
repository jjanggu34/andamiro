## 결함 목록

### 1. 방 비밀번호가 외부에서 읽힐 수 있음 `Critical`

**파일** `src/stores/exchange.js` 265번째 줄

**무슨 문제인가**  
교환일기 방에 비밀번호를 걸어도 실제로는 보호가 되지 않는 상태입니다.  
현재 코드는 방 비밀번호를 서버가 아닌 브라우저에서 확인합니다. 로그인한 사용자라면 누구든 데이터베이스를 직접 조회해서 모든 방의 비밀번호를 평문으로 볼 수 있습니다.

**어떻게 고치나**  
비밀번호 확인을 Supabase Edge Function(서버) 안에서 처리하도록 변경해야 합니다. 현재 `accept-exchange-invitation` 함수처럼 서버가 검증하는 방식으로 `joinRoom`을 교체하면 됩니다.

**오픈 전 필수 수정 항목입니다.**

---

### 2. 교환일기 작성 중 앱을 나가면 오늘 일기가 사라짐 `High`

**파일** `src/views/chat/ResultView.vue` 190번째 줄

**무슨 문제인가**  
감정 분석 결과 화면에서 '교환일기 작성하기'를 선택하면 오늘 일기가 저장되지 않은 채 채팅 데이터가 삭제됩니다. 교환일기 작성 화면으로 이동한 뒤 뒤로가기를 하거나 앱을 종료하면 오늘 감정 일기를 복구할 방법이 없습니다.

**어떻게 고치나**  
교환일기 작성 화면으로 이동하기 전에 오늘 일기를 먼저 저장하도록 순서를 변경합니다. 저장 완료 후에 교환일기 작성으로 넘어가면 됩니다.

---

### 3. 저장 중 네트워크가 느리면 일기가 중복 생성될 수 있음 `Medium`

**파일** `src/stores/diary.js` 79번째 줄

**무슨 문제인가**  
일기 저장 시 10초 안에 응답이 없으면 "저장 시간이 초과됐어요" 에러가 표시됩니다. 그런데 에러가 표시된 뒤에도 실제 저장 요청은 계속 진행됩니다. 11~12초 후 저장이 완료되면 DB에는 일기가 생성된 상태인데 사용자는 에러를 보고 다시 저장 버튼을 누르게 됩니다. 결과적으로 같은 날 일기가 두 개 생성됩니다.

**어떻게 고치나**  
타임아웃이 발생할 때 Supabase 요청도 함께 취소(`AbortController`)하거나, 저장 전에 같은 날짜 기록이 이미 있는지 확인하는 로직을 추가합니다.

---

### 4. 교환일기 방 생성 실패 시 목록에 빈 항목이 추가됨 `Medium`

**파일** `src/stores/exchange.js` 191번째 줄

**무슨 문제인가**  
교환일기 방을 만들 때 서버 응답 형식이 예상과 다를 경우(네트워크 오류, 서버 배포 문제 등), ID가 없는 빈 게시글 항목이 목록 화면에 추가됩니다. 이 항목은 공유하거나 삭제할 수 없고 화면에서 사라지지도 않아 혼란을 줍니다.

**어떻게 고치나**  
서버 응답에서 `room` 데이터가 정상적으로 포함되어 있는지 먼저 확인한 뒤 목록에 추가하도록 코드를 수정합니다.

---

### 5. 알림 클릭 시 의도하지 않은 화면으로 이동될 수 있음 `Medium`

**파일** `src/router/index.js` 91번째 줄

**무슨 문제인가**  
푸시 알림에 포함된 이동 경로(`notificationTarget`)를 아무 검증 없이 그대로 사용합니다. 누군가 조작된 링크를 만들면 앱 안의 임의 화면으로 이동시킬 수 있습니다.

**어떻게 고치나**  
허용된 경로 목록을 정의하고, `notificationTarget` 값이 그 안에 있는 경우에만 이동하도록 처리합니다.

---

### 6. 메인 화면 진입 시 달력 데이터를 두 번 불러옴 `Low`

**파일** `src/views/main/MainView.vue` 117번째 줄

**무슨 문제인가**  
메인 화면이 열릴 때 일기 데이터를 두 곳에서 동시에 요청합니다. 실제 문제가 생기는 경우는 드물지만 불필요한 서버 요청이 발생하고, 네트워크가 느릴 때 두 응답이 엇갈려 표시되는 데이터가 잠깐 이상하게 보일 수 있습니다.

**어떻게 고치나**  
두 곳 중 한 곳에서만 데이터를 불러오도록 중복 호출을 제거합니다.
